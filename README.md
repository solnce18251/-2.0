# Данные по ИТ рынку кандидатов в России 📊

Система автоматического сбора и анализа данных о зарплатных ожиданиях ИТ-кандидатов из открытых источников.

## 🚀 Возможности

- **Аналитика зарплатных ожиданий ИТ-специалистов** — статистика по зарплатам (мин, медиана, макс, персентили)
- **Наполняемость рынка ИТ-специалистов** — количество открытых вакансий по ролям, уровням и городам
- **Парсинг данных** с популярных job-сайтов (hh.ru, Habr Career)
- **Автоматическое определение** ИТ-роли и уровня специалиста (Junior/Middle/Senior/TeamLead)
- **Веб-дашборд** для визуализации статистики
- **Фильтрация** по ролям, уровням, городам и периодам

## 🛠️ Установка

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Настройка

Отредактируйте файл `.env` при необходимости:

```env
# База данных
DATABASE_URL=sqlite:///salary_data.db

# Сервер
HOST=127.0.0.1
PORT=5000

# Парсинг
PARSER_INTERVAL_HOURS=24
MAX_PAGES_PER_SOURCE=10
```

## 📖 Использование

### Быстрый старт

```bash
# 1. Запуск парсинга данных (собирает вакансии по всем ролям)
python run.py parse

# 2. Запуск веб-сервера
python run.py server

# 3. Откройте браузер: http://127.0.0.1:5000
```

### Команды CLI

```bash
# Парсинг всех ролей
python run.py parse

# Парсинг конкретных ролей
python run.py parse --roles python_developer java_spring_developer --cities Москва Санкт-Петербург

# Парсинг одной роли
python run.py parse --role python_developer --city Москва --pages 5

# Запуск сервера
python run.py server --port 5000 --debug

# Просмотр статистики БД
python run.py stats

# Очистка старых данных (старше 90 дней)
python run.py clear --days 90
```

## 🌐 Веб-интерфейс

### Фильтры

- **Период**: последние 30 дней / 12 недель / 6 месяцев / весь период
- **Город**: Москва, Санкт-Петербург, другие города России
- **Уровень**: Junior, Middle, Senior, Team Lead
- **Роли**: выбор одной или нескольких IT-ролей

### Визуализация

1. **Карточки статистики**: P25, медиана, P75, средняя зарплата
2. **Тренд зарплат**: изменение во времени
3. **Распределение по уровням**: сравнение Junior/Middle/Senior/TeamLead
4. **Сравнение по городам**: географическая разница в зарплатах
5. **Сравнение ролей**: детальное сравнение всех IT-специальностей

## 🔌 API Endpoints

## ⚠️ Важные замечания

1. **Парсинг**: Соблюдайте правила источников данных и не делайте слишком частые запросы
2. **Данные**: Зарплаты нормализуются к RUB по приблизительному курсу
3. **Периодичность**: Рекомендуется обновлять данные раз в 24 часа

## 🔧 Расширение

### Добавление новой роли

1. Откройте `config.py`
2. Добавьте роль в словарь `IT_ROLES`:
```python
"new_role": {"name": "Название роли", "category": Category.DEVELOPMENT},
```
3. Добавьте ключевые слова в `ROLE_KEYWORDS`:
```python
"new_role": ["keyword1", "keyword2"],
```

### Добавление нового источника

1. Создайте новый файл парсера в `parsers/`
2. Унаследуйтесь от `BaseParser`
3. Реализуйте метод `parse()`
4. Добавьте парсер в `DataCollector`
